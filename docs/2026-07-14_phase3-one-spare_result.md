# Phase 3 — 스페어 하나 구현 결과

Written at (KST): 2026-07-14 15:30

## 1. 한눈에 보는 요약

- 스페어 보너스(다음 1투구를 더하는 규칙)가 정확히 반영되도록 `score()`를 프레임 단위
  순회 방식으로 바꾸었다.
- `5, 5, 3, 나머지 0` 투구 시 총점이 16으로 정확히 계산됨을 테스트로 확인했다.

## 2. 왜 필요했는가

Phase 1~2의 "핀 총합 반환" 방식은 보너스가 없는 게임에서만 맞는 값을 준다. 실제 볼링
규칙(스페어 보너스)을 반영하려면 프레임 경계와 보너스 투구를 인식하는 로직이 필요하다.

## 3. Before / After

- **Before**: 총점 = 모든 투구 핀 수의 합 (스페어 보너스 미반영).
- **After**: 총점 = 프레임별로 (기본 점수 + 스페어면 다음 1투구 보너스)를 10프레임만큼
  더한 값.

## 4. 대표 시나리오

- 프레임1이 스페어(5,5)이고 다음 투구가 3 → `score() == 16`.
  (`tests/test_game.py::test_one_spare_adds_next_roll_as_bonus`)
- 기존 시나리오(거터 게임, 모두 1핀)도 회귀 없이 계속 통과한다.

## 5. 동작/구조 변화

- `game.py`의 `score()`가 단순 합산에서 "10프레임을 순회하며, 두 투구 합이 10이면
  스페어로 보고 다음 투구를 보너스로 더한다"는 방식으로 바뀌었다.
- 스트라이크(첫 투구 10) 처리는 아직 없다 — 다음 phase에서 다룬다.

## 6. 영향 범위

- `score()`를 호출하는 모든 시나리오에 적용된다. 스트라이크가 섞인 입력은 다음 phase
  전까지는 정확하지 않을 수 있다(테스트로 아직 다루지 않음).

## 7. 제약 / 후속 작업 / 알려진 한계

- 스트라이크, 10번째 프레임 보너스, 퍼펙트 게임은 아직 테스트/구현되지 않았다.

## 8. Lessons

- 특별한 교훈 없음 (예상대로 진행됨).
