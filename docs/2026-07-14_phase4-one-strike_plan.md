# Phase 4 — 스트라이크 하나 구현 계획

Written at (KST): 2026-07-14 15:33

## 1. 작업 목적

Phase 3의 스페어 로직은 "두 투구 합이 10"만 스페어로 판정한다. 첫 투구가 10(스트라이크)
인 경우, 한 번의 투구로 프레임이 끝나고 다음 2투구가 보너스가 되어야 하는데, 아직
반영되지 않았다. 이를 구현한다.

## 2. 현재 상태 (독자 관점)

- `Game.score()`는 스페어(두 투구 합 10)는 처리하지만, 스트라이크(첫 투구가 10)는 별도
  분기가 없어 잘못 계산된다.
- 수동 확인: `10, 3, 5, 나머지 0` → 현재 결과 18, 기대값 26
  (프레임1: 10+3+5보너스=18, 프레임2: 3+5=8, 나머지 0 → 18+8=26).

## 3. 목표 상태

- 첫 프레임이 스트라이크이고 다음 두 투구가 3, 5일 때 총점이 26이 되도록 한다.

## 4. 대표 시나리오

- 투구 순서: `10, 3, 5, 0×16` → `score() == 26`

## 5. 제안하는 변경 접근

- `score()`의 프레임 순회에 스트라이크 분기를 추가한다: 현재 투구가 10이면 해당
  프레임은 1투구만 소비하고, 다음 2투구를 보너스로 더한다. 스트라이크 판정을 스페어
  판정보다 먼저 검사해야 오판을 막을 수 있다.

## 6. 예상 설명 구조

결과는 `docs/2026-07-14_phase4-one-strike_result.md`에 정리한다.

## 7. 모호성 / 가정 / 트레이드오프 / 리스크

- **가정**: 이 phase의 테스트는 10번째 프레임이 아닌 프레임에서의 스트라이크만 다룬다.
  10번째 프레임의 특수 처리는 다음 phase에서 별도로 검증한다.

## 8. 검증 방법

- `PYTHONPATH=. pytest -q`로 새 테스트가 실패(Red) → 통과(Green)하는지 확인하고, 기존
  테스트(거터 게임, 모두 1핀, 스페어)가 계속 통과하는지 함께 확인한다.
