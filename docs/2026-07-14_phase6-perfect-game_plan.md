# Phase 6 — 퍼펙트 게임 구현 계획

Written at (KST): 2026-07-14 15:45

## 1. 작업 목적

CLAUDE.md에 정의된 마지막 요구사항인 퍼펙트 게임(12번 모두 스트라이크 → 총점 300)이
정확히 계산되는지 확인한다. 이는 지금까지 구현한 모든 규칙(스트라이크, 10번째 프레임
보너스)이 함께 맞물려 동작하는지 검증하는 종합 시나리오다.

## 2. 현재 상태 (독자 관점)

- `Game.score()`는 스트라이크, 스페어, 10번째 프레임 보너스를 모두 처리한다.
- 수동 확인 결과, 12번 연속 스트라이크(퍼펙트 게임) 투구 시 이미 300점이 계산된다.

## 3. 목표 상태

- 12번의 투구가 모두 스트라이크(10)일 때 `score()`가 `300`을 반환함을 테스트로
  명세해 둔다.

## 4. 대표 시나리오

- 투구 순서: `10 × 12` → `score() == 300`

## 5. 제안하는 변경 접근

- 테스트만 추가한다. Phase 4의 스트라이크 분기와 고정 10회 순회 lookahead 구조가
  이미 이 케이스를 만족시킬 것으로 예상된다 (Phase 2, 5와 동일한 패턴).

## 6. 예상 설명 구조

결과는 `docs/2026-07-14_phase6-perfect-game_result.md`에 정리하며, 실제로 구현 변경이
필요했는지 여부를 투명하게 기록한다.

## 7. 모호성 / 가정 / 트레이드오프 / 리스크

- **예상되는 예외 상황**: Phase 2, 5와 마찬가지로 구현 변경 없이 즉시 통과할 가능성이
  높다. 실제 실행 결과에 따라 결과 문서의 서술을 확정한다.

## 8. 검증 방법

- `PYTHONPATH=. pytest -q`로 새 테스트를 포함한 전체 스위트가 통과하는지 확인한다.
