# Phase 6 — 퍼펙트 게임 구현 결과

Written at (KST): 2026-07-14 15:47

## 1. 한눈에 보는 요약

- 퍼펙트 게임(12번 모두 스트라이크 → 총점 300)을 테스트로 명세해 두었다.
- Phase 2, 5와 같은 패턴으로, **구현 변경 없이 테스트 추가만으로 즉시 통과(Green)했다.**
- 이것으로 CLAUDE.md에 정의된 5개 요구사항(기본 점수, 스페어, 스트라이크, 10번째
  프레임, 퍼펙트 게임)이 모두 테스트로 명세되고 구현되었다.

## 2. 왜 필요했는가

퍼펙트 게임은 스트라이크·10번째 프레임 보너스 규칙이 모두 맞물려 동작하는지 확인하는
종합 시나리오이자, 이 kata의 가장 잘 알려진 대표 사례(총점 300)다.

## 3. Before / After

- **Before**: 퍼펙트 게임 시나리오가 테스트로 명세되어 있지 않았다.
- **After**: 12연속 스트라이크 시 총점 300이 명세되어 있다. `game.py`는 변경되지
  않았다.

## 4. 대표 시나리오

- 12번의 투구가 모두 스트라이크 → `score() == 300`.
  (`tests/test_game.py::test_perfect_game_scores_three_hundred`)

## 5. 동작/구조 변화

- 없음. `game.py`는 Phase 4와 동일하다.

## 6. 영향 범위

- 없음(구현 변경이 없으므로 기존 동작에 영향 없음).

## 7. 제약 / 후속 작업 / 알려진 한계

- CLAUDE.md에 정의된 범위(기본 점수 → 스페어 → 스트라이크 → 10번째 프레임 → 퍼펙트
  게임) 내의 요구사항은 모두 충족했다. 입력값 검증, 중간 프레임 조회 등은 여전히
  범위 밖이다(CLAUDE.md 1절 참고).

## 8. Lessons

- **교훈**: Bowling Kata는 스트라이크/스페어 분기와 "정확히 10회 순회" 구조만 갖추면,
  10번째 프레임 특수 규칙과 퍼펙트 게임까지 추가 코드 없이 자연스럽게 만족된다. 이는
  이 kata가 "일반화된 최소 구현(Obvious Implementation)"의 힘을 보여주는 대표적인
  예시로 잘 알려져 있다.
