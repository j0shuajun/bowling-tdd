# Phase 1 — 거터 게임(기본 점수) 구현 계획

Written at (KST): 2026-07-14 14:58

## 1. 작업 목적

Bowling Game Kata의 첫 단계로, `Game` 클래스의 뼈대(`roll(pins)`, `score()`)를 세우고
"보너스가 전혀 없는 가장 단순한 경우"부터 점수 계산이 올바르게 동작하는지 확인한다. 아직
`game.py`가 존재하지 않으므로, 이 phase는 클래스/메서드의 최소 형태를 만드는 것이 목표다.

## 2. 현재 상태 (독자 관점)

- 프로젝트에 `game.py`가 없고 `tests/` 디렉토리도 비어 있다.
- `Game` 클래스, `roll`, `score`가 아직 아무것도 정의되지 않아 어떤 투구를 넣어도 점수를
  계산할 수 없다.

## 3. 목표 상태

- `Game()`을 생성하고 `roll(pins)`을 20번(모든 프레임에서 2번씩, 총 10프레임) 호출한 뒤
  `score()`를 호출하면, 투구가 전부 0핀(거터)일 때 총점 0을 반환한다.
- 이 phase는 스페어/스트라이크 등 보너스 로직 없이 "투구를 기록하고, 그 합을 반환"하는
  가장 단순한 경로만 다룬다.

## 4. 대표 시나리오

- **거터 게임**: 20번의 투구가 모두 0핀 → `score()` 결과는 `0`.

## 5. 제안하는 변경 접근

- `game.py`에 `Game` 클래스를 추가한다.
- `roll(pins)`은 투구 기록을 내부 리스트에 누적한다.
- `score()`는 (이 phase 범위에서는) 누적된 투구 핀 수의 합을 반환한다. 스페어/스트라이크
  보너스는 이후 phase에서 다룬다.

## 6. 예상 설명 구조

이번 phase는 코드 변경이므로 별도의 사용자 문서(README 등) 갱신은 필요 없다. 결과는
`docs/2026-07-14_phase1-gutter-game_result.md`에 Before/After 형태로 정리한다.

## 7. 모호성 / 가정 / 트레이드오프 / 리스크

- **가정**: `roll(pins)`은 매 투구마다 정확히 한 번씩, 순서대로 호출된다고 가정한다
  (CLAUDE.md에 따라 입력값 검증은 범위 밖).
- **가정**: 이 phase에서 `score()`의 구현은 "핀 총합 반환"이라는 최소 형태로 두고, 스페어/
  스트라이크 보너스 계산은 다음 phase에서 확장한다 (Fake it → 점진적 구현 전략).
- **리스크**: 단순 합산 구현이라 이후 phase(스페어/스트라이크)에서 로직을 상당 부분
  재작성해야 할 수 있다. 이는 Kata의 의도된 진행 방식이므로 허용한다.
