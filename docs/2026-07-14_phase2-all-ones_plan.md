# Phase 2 — 모두 1핀(보너스 없는 일반 프레임) 구현 계획

Written at (KST): 2026-07-14 15:12

## 1. 작업 목적

Phase 1에서 만든 "거터 게임" 다음 단계로, 보너스가 없는 일반적인 프레임 조합에서도 점수
합산이 올바른지 "모두 1핀" 시나리오로 명세를 하나 더 남긴다.

## 2. 현재 상태 (독자 관점)

- `game.py`의 `Game.score()`는 지금까지 던진 핀 수의 합을 그대로 반환한다 (Phase 1 결과).
- 스페어/스트라이크가 없는 일반 게임에서는, 총점이 정의상 "쓰러뜨린 핀 수의 총합"과 같다.
  즉 현재 구현이 이미 이 케이스를 만족시킬 가능성이 있다.

## 3. 목표 상태

- 20번의 투구가 모두 1핀일 때 `score()`가 `20`을 반환함을 테스트로 명세해 둔다.

## 4. 대표 시나리오

- **모두 1핀**: 20번의 투구가 모두 1핀 → `score()` 결과는 `20`.

## 5. 제안하는 변경 접근

- 테스트만 추가한다. Phase 1의 "핀 총합 반환" 구현이 보너스 없는 모든 프레임 조합에 대해
  이미 수학적으로 올바르므로(보너스가 없으면 총점 = 핀 총합), 구현 변경 없이 통과할 것으로
  예상된다.

## 6. 예상 설명 구조

이번 phase도 코드 변경 문서로, 결과는
`docs/2026-07-14_phase2-all-ones_result.md`에 정리한다.

## 7. 모호성 / 가정 / 트레이드오프 / 리스크

- **예상되는 예외 상황**: 이 테스트는 "Red(실패) → Green(통과)" 쌍을 이루지 못하고, 작성한
  즉시 통과(Green)할 가능성이 높다. 보너스 없는 케이스는 Phase 1 구현이 이미 정답을
  생성하기 때문이다.
- 이 경우, 이 phase는 "테스트만 추가되고 구현 변경이 없는 phase"로 결과 문서에 그 사실을
  투명하게 기록한다 (거짓으로 실패를 유도하거나 구현을 억지로 바꾸지 않는다).
- 실제로 실행해 확인한 뒤, 통과 여부에 따라 결과 문서의 서술을 확정한다.

## 8. 검증 방법

- `PYTHONPATH=. pytest -q`로 새 테스트를 포함한 전체 스위트가 통과하는지 확인한다.
- 테스트 추가 직후(구현 변경 전) 이미 통과하는지, 즉시 실행해 확인한다.
